# LAKOTA HOUSE モックサイト 仕様書

## 1. 機能要件一覧

| ID | 機能名 | 対象ページ | 優先度 | 概要 |
|----|--------|-----------|--------|------|
| F01 | ハンバーガーメニュー | 全ページ | 高 | SP時にグローバルナビを開閉する |
| F02 | ヘッダースクロール変化 | 全ページ | 高 | スクロールでヘッダー背景を透明→白に変化 |
| F03 | スクロールフェードイン | 全ページ | 中 | 要素がビューポートに入った際にフェードイン表示 |
| F04 | ヒーローアニメーション | index.html | 中 | ページロード時にヒーローテキストをフェードイン |
| F05 | ギャラリーフィルタ | gallery.html | 高 | カテゴリボタンで画像の表示/非表示を切り替え |
| F06 | ギャラリーモーダル | gallery.html | 高 | 画像クリックで拡大表示、前後ナビ、閉じる機能 |
| F07 | フォームバリデーション | contact.html | 高 | リアルタイム＋送信時の入力チェック |
| F08 | サンクスメッセージ表示 | contact.html | 高 | 送信成功後にフォームを非表示、サンクスを表示 |
| F09 | スムーススクロール | 全ページ | 低 | アンカーリンクでスムーススクロール |

---

## 2. 非機能要件

| 項目 | 要件 |
|------|------|
| 動作環境 | ローカルファイル（`file://` プロトコル）で動作すること |
| 外部依存 | サーバー不要。Google Fonts・Font Awesome のみ CDN 利用（オンライン時） |
| フレームワーク | CSS フレームワーク不使用（純粋な CSS）、jQuery 不使用（Vanilla JS） |
| レスポンシブ | PC（1200px以上）/ タブレット（768〜1024px）/ スマホ（768px以下）対応 |
| セマンティクス | header, nav, main, section, footer, article 等のセマンティクスタグを使用 |
| アクセシビリティ | aria-label, role, alt テキストを適切に設定 |
| パフォーマンス | 外部画像は `placehold.co` のプレースホルダー使用、`loading="lazy"` 設定 |

## 4. バリデーション仕様一覧

| フィールド | チェック内容 | タイミング | エラーメッセージID |
|-----------|------------|-----------|-----------------|
| name | 空欄チェック | blur・送信時 | #name-error |
| email | 空欄チェック | blur・送信時 | #email-error |
| email | メール形式チェック | blur・送信時 | #email-error |
| inquiry-type | 未選択チェック | blur・送信時 | #inquiry-type-error |
| message | 空欄チェック | blur・送信時 | #message-error |
| privacy | 未チェックチェック | 送信時 | #privacy-error |

**リアルタイムバリデーション:**
- `blur` イベントでバリデーションを実行
- 一度エラーになったフィールドは `input` イベントでも再チェック（エラーがあるうちは）
